Feature Spec

Confirm Booking CTA

The primary call-to-action button at the bottom of the booking form. Full-width navy gradient button. Includes disabled and loading states.

Default
Disabled (incomplete form)
Loading (submitting)

Design notes

  • Full-width button spanning the form column
  • Background: navy gradient linear-gradient(135deg, #001e40, #003366)
  • Text: white, 14px, weight 700, 0.02em tracking
  • Padding: 16px 24px, radius 12px
  • Shadow: 0 14px 28px rgba(0,30,64,0.20)
  • Hover: opacity 0.92
  • Active: scale(0.985) press effect
  • Disabled state: #e8e8e6 bg, #737780 text, no shadow, cursor not-allowed
  • Loading state: text hidden, 20px white spinner centered
  • Disabled state is a visual spec only — the current button is always enabled and validates on submit. Adding disabled-until-valid logic would be a functional change and is out of scope for this design-only reskin. Apply disabled styling only to the existing isSubmitting state
  • Design change only — no new functionality

Implementation context (post spec 01–06 reskin)

  • The current button (line ~1392) uses --color-* tokens: background var(--color-brand)/var(--color-brand-dim), color var(--color-surface-void), borderRadius var(--radius-lg). Replace with --al-* gradient + shadow
  • Current loading state uses text change ("Booking…") + opacity 0.7. The spinner requires a @keyframes rule — add to globals.css (precedent: .al-input-wrap:focus-within was already added for spec 06's focus ring)
  • The gradient and shadow values match the nav CTA (spec 01, implemented): var(--al-gradient-cta) and 0px 14px 28px rgba(0,30,64,0.20)
  • The button is the last element inside <form onSubmit={handleSubmit} className="space-y-7"> — vertical spacing from the comm prefs above is 28px via that Tailwind class
  • Between the comm prefs and the button, there is an error block (lines ~1383-1390) using --color-error-border, --color-error-subtle, --color-error tokens. This error block is NOT part of the CTA spec — restyle it alongside the button or leave as a separate TODO
  • The Stripe payment step also has buttons (Pay now, Back, Cancel) that still use --color-* tokens — those are out of scope for this spec and remain unchanged
