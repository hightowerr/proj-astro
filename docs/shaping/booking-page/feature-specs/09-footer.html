Feature Spec

Footer

Minimal footer for the public booking page. Shows the Astro brand mark centered, with a "Powered by" attribution line below. Signals trust and legitimacy.

Powered by Astro · Protected booking

Design notes

  • Centered layout, 32px padding
  • Top border: 1px solid rgba(195,198,209,0.20) — hairline separator from content
  • Brand mark: 28px navy square, radius 8px, Material Symbol icon centered
  • Brand name: 16px, weight 800, 0.04em tracking, uppercase, navy
  • Attribution text: 13px, muted (#737780), "Powered by Astro · Protected booking"
  • 12px gap between brand mark and attribution
  • "Protected booking" conveys security / trust signal to the customer
  • Design change only — no new functionality

Implementation context (post V1–V3 reskin)

  • The shared SiteFooter no longer renders on /book routes — RouteChrome excludes "/book" as of V1
  • The booking layout lives at src/app/book/layout.tsx. It currently renders BookingNav + {children} with no footer
  • This footer component should be added to src/app/book/layout.tsx, rendered after {children}
  • The brand mark in the nav (spec 01, implemented) uses the same Material Symbol + navy square pattern at 32px — this footer version is 28px, same style but smaller
