Major — Forbidden Tokens in Shared Component IMPLEMENTED — Wave 1 Slice 2 (2026-06-19)

#5 — Button Uses Deep Ledger Tokens

File: src/components/ui/button.tsx · Shared Component · Wave 1

Dependencies: None — uses existing --al-* tokens already in production

Problem

The outline variant of the Button component references forbidden Deep Ledger --color-* tokens. Since Button is used on every page, this creates a system-wide design system violation.

/* Current outline variant */
"border shadow-xs
 [background:var(--color-surface-elevated)]
 [border-color:var(--color-border-medium)]
 [color:var(--color-text-primary)]
 hover:[background:var(--color-surface-float)]
 hover:[border-color:var(--color-border-strong)]"

Fix

Replace with Atelier Light equivalents:

Deep Ledger tokenAL replacement
--color-surface-elevated--al-surface-container-lowest
--color-border-medium--al-outline-variant
--color-text-primary--al-on-surface
--color-surface-float--al-surface-container
--color-border-strong--al-outline
/* Fixed outline variant */
"border shadow-xs
 [background:var(--al-surface-container-lowest)]
 [border-color:var(--al-outline-variant)]
 [color:var(--al-on-surface)]
 hover:[background:var(--al-surface-container)]
 hover:[border-color:var(--al-outline)]"

Also check all other variants in the file for any --color-* references and replace accordingly.
