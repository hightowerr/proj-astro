#13 — Payment Policy Page: Convert Inline Styles to Tailwind

File: src/app/app/settings/payment-policy/page.tsx · Page Conversion · Wave 4

Dependencies: #04 — needs .al-page, .al-eyebrow, .al-page-title, .al-section-title, .al-card

Problem

The payment policy settings page uses ~35 inline style={{}} objects, including a complex TiersExplainerCard component with extensive inline styling:

/* Page wrapper */
style={{ minHeight: '100vh', background: 'var(--al-surface)',
         fontFamily: 'var(--al-font)' }}

/* Content container */
style={{ maxWidth: 1080, margin: '0 auto',
         padding: '32px 48px' }}

/* TiersExplainerCard — 3-column grid entirely inline */
style={{ display: 'grid', gridTemplateColumns: 'repeat(3, 1fr)', gap: 18 }}

/* Hardcoded hex colors mixed with CSS variables */
"#001e40", "#43474f"
background: 'linear-gradient(135deg, #001e40, #003366)'

/* Material Symbols with inline fontVariationSettings */
style={{ fontFamily: 'Material Symbols Outlined',
         fontVariationSettings: "'FILL' 0, 'wght' 400" }}

Fix

  • Page wrapper → className="al-page"
  • Content container → className="max-w-5xl mx-auto" (1024px, close to 1080px)
  • Breadcrumb → className="al-eyebrow"
  • Page title → className="al-page-title"
  • Section titles → className="al-section-title"
  • Card containers → className="al-card p-6"
  • TiersExplainerCard grid → className="grid grid-cols-3 gap-[18px]"
  • Tier badge colors → AL status tokens
  • CTA gradient button → al-gradient-cta + Tailwind classes
  • Material Symbols icons → className="material-symbols-outlined" (class handles font-variation-settings). Note: inline styles also set custom fontSize values (13px, 14px, 15px) that differ from the class default (20px). Add Tailwind size overrides (e.g., text-[15px]) when removing inline styles, or icon sizes will revert to 20px.
  • Hardcoded hex → AL utility classes

Target: Zero style={{}} props.
