#9 — Appointments Page: Convert Inline Styles to Tailwind

File: src/app/app/appointments/page.tsx · Page Conversion · Wave 4

Dependencies: #04 — needs .al-page, .al-eyebrow, .al-page-title, .al-section-title, .al-lede, .al-card, .al-mono

Problem

The appointments page (~365 lines) uses ~63 inline style={{}} objects instead of Tailwind classes. Every element has hardcoded pixel values and hex colors:

/* Root wrapper */
style={{ fontFamily: "var(--al-font)", background: "var(--al-background)",
         minHeight: "100vh", padding: "32px 48px",
         display: "flex", flexDirection: "column", gap: 28 }}

/* Eyebrow pattern repeated 5+ times */
style={{ fontSize: 10, fontWeight: 800, letterSpacing: ".2em",
         textTransform: "uppercase", color: "#43474f", opacity: .6 }}

/* Hardcoded hex colors throughout */
"#001e40"  /* should be var(--al-primary) */
"#43474f"  /* should be var(--al-on-surface-variant) */
"#0e7a55"  /* should be var(--al-status-positive) */
"#c97a2a"  /* should be var(--al-status-caution) */
"#a8294a"  /* should be var(--al-status-negative) */

/* Card containers with inline box model */
style={{ background: "#fff", borderRadius: 24,
         overflow: "hidden", boxShadow: "0 20px 40px rgba(26,28,27,.04)" }}

This is the most inline-heavy page in the app and the primary source of visual inconsistency.

Fix

Convert all inline styles to Tailwind classes and AL utility classes. Key conversions:

  • Root wrapper → className="al-page"
  • All eyebrow patterns → className="al-eyebrow"
  • Page title → className="al-page-title"
  • Section titles → className="al-section-title"
  • Lede/description text → className="al-lede"
  • Card containers → className="al-card"
  • Hardcoded #001e40 → text-al-primary
  • Hardcoded #43474f → text-al-on-surface-variant
  • Outcome stat numbers → add al-mono class
  • Status colors → text-[var(--al-status-positive)] etc.
  • Grid layouts → grid grid-cols-3 gap-3.5
  • Inline padding/margin → Tailwind spacing classes (p-6, gap-7, etc.)

The RECOVERY_STATUS color object should use AL status token references instead of hardcoded hex tuples.

Target: Zero style={{}} props remaining. All styling via Tailwind + AL classes.
