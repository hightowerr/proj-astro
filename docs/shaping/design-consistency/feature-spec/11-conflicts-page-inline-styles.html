#11 — Conflicts Page: Convert Inline Styles to Tailwind

File: src/app/app/conflicts/page.tsx · Page Conversion · Wave 4

Dependencies: #04 — needs utility classes (.al-page, .al-eyebrow, .al-page-title, .al-card)

Problem

The conflicts page uses ~14 inline style={{}} objects. Note: the child component src/components/conflicts/conflicts-ledger.tsx contains ~67 additional inline styles and should be scoped into this migration or handled as a companion spec. Same structural issues as the appointments and customers pages:

  • Root wrapper with inline fontFamily, padding, background, minHeight, display, flexDirection, gap
  • Hardcoded hex colors: #001e40, #43474f, #0e7a55, #737780
  • Inline gradient backgrounds and shadows on buttons
  • Page title at fontSize: 40px instead of canonical 44px
  • Inline eyebrow patterns with fontSize: 10px, letterSpacing: ".2em"
  • Custom Icon component with Material Symbols rendered via inline styles

Fix

Same conversion pattern as Specs #9 and #10:

  • Root wrapper → className="al-page"
  • Page title (40px) → className="al-page-title" (standardize to 44px)
  • Eyebrows → className="al-eyebrow"
  • Description → className="al-lede"
  • Cards → className="al-card"
  • Back link → Tailwind flex + AL text colors
  • CTA buttons → al-gradient-cta + Tailwind padding/radius or the shared Button component
  • All hardcoded hex → AL utility classes

Target: Zero style={{}} props.
