#8 — Booking & Service Components Use Deep Ledger Tokens

Components: src/components/booking/, src/components/services/, src/components/manage/, src/components/conflicts/ · Shared Components · Wave 2

Dependencies: #07 (soft) — references #07's mapping table for consistency, not a code dependency

Problem

Multiple feature components reference forbidden Deep Ledger --color-* tokens:

  • booking/booking-form.tsx — uses --color-surface-*, --color-brand, --color-text-*, --color-border-*
  • booking/service-selector.tsx — uses .service-card class (defined with --color-* tokens), --color-brand-border
  • services/event-type-list.tsx — uses --color-surface-raised, --color-border-default, --color-text-*
  • services/event-type-form.tsx — uses --color-* tokens for form elements
  • manage/manage-booking-view.tsx — uses --color-surface-*, --color-brand, --color-text-*
  • conflicts/conflict-alert-banner.tsx — uses --color-* tokens for the alert styling

Fix

Apply the standard token mapping to all six files. Use the same mapping table as Spec #7:

Deep Ledger tokenAL replacement
--color-border-default--al-outline-variant
--color-border-medium--al-outline-variant
--color-border-hairline--al-ghost-border
--color-surface-raised--al-surface-container-lowest
--color-surface-elevated--al-surface-container

--color-text-secondary--al-on-surface-variant

--color-brand-border--al-hairline-strong (hover) / --al-hairline-rest (rest) — AL uses neutral hairlines, not brand-tinted borders (precedent set by Wave 1 Spec #07)

--color-brand-subtlergba(0, 30, 64, 0.08)

For the .service-card CSS class usage in service-selector.tsx, switch to .al-service-card (already defined in globals.css ~line 646).

Note (post-Wave 1): The --al-hairline base token (0.20 opacity) is not defined in globals.css; use --al-ghost-border instead (same value). The --color-brand-border mapping was changed from brand-tinted to neutral hairlines per Wave 1 Spike A findings.
