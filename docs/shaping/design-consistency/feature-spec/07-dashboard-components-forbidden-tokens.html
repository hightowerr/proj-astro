Major — Forbidden Tokens in Dashboard Components IMPLEMENTED — Wave 1 Slice 3 (2026-06-19)

#7 — Dashboard Components Use Deep Ledger Tokens

Components: src/components/dashboard/ · Shared Components · Wave 1

Dependencies: None — uses existing --al-* and --al-status-* tokens already in production

Problem

Five dashboard components use forbidden --color-* Deep Ledger tokens or hardcoded non-AL colors:

shop-overview-card.tsx

Entirely styled with --color-border-default, --color-surface-raised, --color-text-primary, --color-text-tertiary, --color-brand, --color-brand-hover, --color-text-inverse, --color-surface-base.

confirmation-status-badge.tsx

Uses hardcoded Tailwind dark-theme colors instead of AL status tokens:

bg-amber-500/15 text-amber-300    /* for pending */
bg-emerald-500/15 text-emerald-300 /* for confirmed */
bg-rose-500/15 text-rose-300       /* for expired */

success-banner.tsx

Uses --color-brand, --color-surface-raised, --color-text-primary.

copy-button.tsx

Uses --color-brand, --color-text-secondary.

booking-management-choice.tsx

Uses --color-surface-raised, --color-border-default, --color-text-*, --color-brand.

Fix

Apply the standard token mapping across all five files:

Deep Ledger / hardcodedAL replacement
--color-border-default--al-outline-variant
--color-surface-raised--al-surface-container-lowest
--color-surface-base--al-background
--color-text-primary--al-on-surface
--color-text-secondary--al-on-surface-variant
--color-text-tertiary--al-on-surface-variant
--color-text-inverse--al-on-primary
--color-brand--al-primary
--color-brand-hover--al-primary-container
bg-amber-500/15 text-amber-300bg-[var(--al-status-caution-bg)] text-[var(--al-status-caution)]
bg-emerald-500/15 text-emerald-300bg-[var(--al-status-positive-bg)] text-[var(--al-status-positive)]
bg-rose-500/15 text-rose-300bg-[var(--al-status-negative-bg)] text-[var(--al-status-negative)]
