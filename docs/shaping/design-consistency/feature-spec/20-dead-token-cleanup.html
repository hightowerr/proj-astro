#20 — Remove Dead Deep Ledger Tokens

File: src/app/globals.css · Cleanup · Wave 5

Dependencies: #05 – #19 — all component and page migrations must be complete before removing tokens

Problem

After Specs #5–#19 migrate all pages and components away from Deep Ledger tokens, the following sections in globals.css become dead code. Line numbers updated 2026-06-19 post-Wave 1 (added ~60 lines of AL tokens):

  • --color-surface-* dark surface tokens (~lines 16–21)
  • Backwards compat aliases --color-bg-dark-* (~lines 24–26)
  • --color-text-* dark text tokens (~lines 28–35)
  • --color-brand-* teal brand tokens (~lines 38–48)
  • --color-accent-* amber accent tokens (~lines 51–58)
  • --color-tier-* dark tier tokens (~lines 82–92)
  • --color-border-* dark border tokens (~lines 95–99)
  • --shadow-* dark shadows (~lines 102–107)
  • .dark {} block (~lines 455–487)
  • Deep Ledger utility classes: .surface-*, .card-glass, .glow-brand, .skeleton, .service-card, .service-card-arrow, .focus-brand, .tier-dot-*, .status-pill (~lines 517–603). Also includes .font-display, .font-body, .font-mono utility classes (~lines 517–519) which now point to AL tokens but retain Deep Ledger-era names.

Post-Wave 2 note: Spec #08 is now implemented — service-selector.tsx uses .al-service-card (line 94). The legacy .service-card class and its .service-card:hover .service-card-arrow rule are now dead CSS. However, the .service-card-arrow class is still referenced by service-selector.tsx (line 126) for the arrow <span> — its hover-reveal is currently non-functional because the parent changed from .service-card to .al-service-card. Either migrate the hover rule to .al-service-card:hover .service-card-arrow or remove the arrow animation. .al-service-card (~line 648) must NOT be removed.

Fix

  1. Verify no remaining references: rg "color-surface|color-text|color-brand|color-border|color-tier|color-accent" src/ should return zero matches.
  2. Remove all dead token declarations listed above from @theme inline.
  3. Remove the .dark {} block entirely.
  4. Remove all Deep Ledger utility classes.
  5. Keep the shadcn compatibility aliases (--primary, --muted, --background, etc.) since they're mapped to AL values and used by shadcn components.
  6. Keep the --color-success, --color-warning, --color-error semantic tokens only if still referenced; otherwise remove in favor of --al-status-*.

Deferred: Landing Page Components

The landing page (hero-section.tsx, site-header.tsx, site-footer.tsx) still uses Deep Ledger tokens for an intentionally dark marketing aesthetic. Converting these is a visual redesign and should be handled in a separate spec/PR. Do not remove tokens that are still referenced by landing page components until that work is complete.
