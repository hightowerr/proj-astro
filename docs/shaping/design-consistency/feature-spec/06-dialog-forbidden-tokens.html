#6 — Dialog Uses Deep Ledger Tokens

File: src/components/ui/dialog.tsx · Shared Component · Wave 2

Dependencies: #03 (implemented) — --al-shadow-menu token already exists in globals.css

Problem

The Dialog component uses inline style={{}} with forbidden Deep Ledger tokens for its background, border, shadow, close button, and description text:

/* DialogContent */
style={{
  background: "var(--color-surface-raised)",       // FORBIDDEN
  border: "1px solid var(--color-border-default)",  // FORBIDDEN
  boxShadow: "var(--shadow-lg)",                    // FORBIDDEN
}}

/* DialogPrimitive.Close */
style={{ color: "var(--color-text-tertiary)" }}     // FORBIDDEN

/* DialogDescription */
style={{ color: "var(--color-text-secondary)" }}    // FORBIDDEN

Fix

Replace all forbidden token references:

Deep Ledger tokenAL replacement
--color-surface-raised--al-surface-container-lowest
--color-border-default--al-outline-variant
--shadow-lg--al-shadow-menu
--color-text-tertiary--al-on-surface-variant
--color-text-secondary--al-on-surface-variant
/* Fixed DialogContent */
style={{
  background: "var(--al-surface-container-lowest)",
  border: "1px solid var(--al-outline-variant)",
  boxShadow: "var(--al-shadow-menu)",
}}

/* Fixed close & description */
style={{ color: "var(--al-on-surface-variant)" }}
