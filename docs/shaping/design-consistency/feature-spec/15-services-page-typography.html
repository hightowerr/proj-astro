Major — Oversized Typography

#15 — Services Page: Title Size 72px → 44px

File: src/app/app/settings/services/page.tsx · Typography Alignment · Wave 4

Dependencies: #04 — needs .al-page-title, .al-eyebrow, .al-lede

Problem

The services settings page has a page title of text-5xl md:text-7xl which renders at 72px on desktop — nearly double the canonical 44px page title. This makes the services page look like a completely different app from every other settings page.

/* Current — way too large */
<h1 className="text-5xl md:text-7xl font-extrabold tracking-tighter leading-[0.9]">
  Services
</h1>

Also uses text-on-surface-variant (missing al- prefix) and has an inline style={{ color: "var(--al-primary)" }} on the breadcrumb.

Fix

  • Page title: text-5xl md:text-7xl font-extrabold tracking-tighter → al-page-title
  • Breadcrumb: Replace text-[10px] font-extrabold uppercase tracking-[0.2em] with al-eyebrow
  • Breadcrumb active item: Replace inline style={{ color: "var(--al-primary)", opacity: 1 }} with className="text-al-primary opacity-100"
  • Subtitle: Replace text-lg font-medium with al-lede
  • Color class: text-on-surface-variant → text-al-on-surface-variant
  • Padding: Align to canonical py-8 px-12
