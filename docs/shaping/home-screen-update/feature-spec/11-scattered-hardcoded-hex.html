#11 — Scattered Hardcoded Hex

Component: atelier-dashboard.tsx · Various Elements · Implemented 2026-06-19

Problem

Multiple hardcoded hex values appear throughout the component instead of DS tokens:

Hardcoded HexMaps To TokenLocationTailwind Utility
#2a170f--al-on-secondary-fixedStep pill texttext-al-on-secondary-fixed
#ffdbcf--al-secondary-fixedStep pill bgbg-al-secondary-fixed
#fdd8cb--al-secondary-containerBook-first icon bg, avatar bg (×3)bg-al-secondary-container
#785c53--al-on-secondary-containerBook-first icon text— no theme mapping
#74584f--al-secondaryAvatar initials text (×3)text-al-secondary
#93000a--al-on-error-containerError pill texttext-al-on-error-container
#ffdad6--al-error-containerError pill bgbg-al-error-container
Resolved: #fff8f5 and #9a6f64 were off-palette and have already been removed by the Spec #2 implementation (Team card switched to white surface with navy text tokens).

Fix

Replace each hex literal with its Tailwind utility or var() inline style. Six of seven tokens have @theme mappings and can use Tailwind classes directly. One requires an inline style or a theme mapping prerequisite.

/* Tailwind utility (6 of 7 tokens) */
bg-[#ffdbcf]  →  bg-al-secondary-fixed
text-[#2a170f] →  text-al-on-secondary-fixed
bg-[#fdd8cb]  →  bg-al-secondary-container
text-[#74584f] →  text-al-secondary
bg-[#ffdad6]  →  bg-al-error-container
text-[#93000a] →  text-al-on-error-container

/* Inline style (no theme mapping) */
text-[#785c53] →  style={{ color: 'var(--al-on-secondary-container)' }}
/* or add --color-al-on-secondary-container to @theme inline first */
Prerequisite: --al-on-secondary-container (#785c53) has no --color-al-on-secondary-container entry in the @theme inline block of globals.css. Either add the mapping or use an inline style for this single value.
