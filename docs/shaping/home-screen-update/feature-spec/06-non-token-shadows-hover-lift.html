Moderate — Token / Scale Violation

#6 — Non-Token Shadows + Hover-Lift on Cards

Component: atelier-dashboard.tsx · All Cards & Icon Tiles

Problem

Shadows are bespoke values instead of design tokens:

  • shadow-[0px_10px_30px_rgba(26,28,27,0.04)]
  • shadow-[...0.03]

The DS has exactly one card float shadow: 0 20px 40px rgba(26,28,27,.04) (--al-shadow-float).

Additionally: The essentials cards add hover:shadow-[...] and icons use group-hover:scale-110. The DS says no hover-lift on cards and motion is deliberately still.

Fix

  1. Use --al-shadow-float on all sheet-level cards.
  2. Remove the hover:shadow-[...] swap classes.
  3. Remove group-hover:scale-110 from icons.
  4. The sanctioned card hover is a background-shift to var(--al-surface-container).
/* Before */
<div className="shadow-[0px_10px_30px_rgba(26,28,27,0.04)]
  hover:shadow-[0px_14px_36px_rgba(26,28,27,0.08)]">
  <span className="group-hover:scale-110" />
</div>

/* After */
<div className="shadow-[var(--al-shadow-float)]
  hover:bg-[var(--al-surface-container)]">
  <span /> <!-- no scale animation -->
</div>
