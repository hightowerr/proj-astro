# Sanity CMS Content Guidelines

## Overview
These guidelines are for non-technical stakeholders (Marketing, Operations) to manage the content of the **Astro** platform using Sanity CMS.

## 1. Landing Page Content
The landing page is composed of several blocks. Each can be edited independently in Sanity Studio.

### Hero Section
- **Headline**: Keep it pain-focused (e.g., "No-shows cost you £5,000 a year. Astro fixes it.").
- **Sub-headline**: 1-2 sentences explaining the "How".
- **CTA Text**: Default is "Book a Demo".
- **Hero Image**: Use high-quality dark-themed salon photos (min 1920x1080).

### Features Section
Each feature block includes:
- **Title**: Action-oriented (e.g., "Recover 100% of No-Show Loss").
- **Description**: Concise and focused on the benefit, not the tool.
- **Float Cards**: Update the statistics (e.g., "94% show-up rate") to reflect real product data.
- **Image**: Visual representation of the feature (mockups or high-quality photos).

### Pricing Section
- **Plan Name**: Default is "Astro Pro".
- **Monthly Price**: Integer value (e.g., 49).
- **Annual Discount**: Percentage value (e.g., 20%).
- **Features List**: Checkable items that appear in the pricing card.

### FAQ Section
- **Question**: Common customer concerns.
- **Answer**: Clear, reassuring, and concise (max 3 sentences).
- **Category**: (Optional) Group by "Payments", "Setup", "Policy", etc.

## 2. Global Settings
- **Site Metadata**: Title and description for SEO.
- **Support Contact**: Email and phone number for the footer.
- **Social Links**: URLs for Instagram, Twitter, etc.

## 3. Best Practices for Content
- **Consistency**: Use the "Teal + Coral" brand voice (professional, bold, reliable).
- **Tone**: Professional yet approachable. Avoid technical jargon like "UUID" or "ORM".
- **Imagery**: Prefer dark-themed images to match the "Dark SaaS" aesthetic. Avoid cluttered photos.
- **Formatting**: Use bold text sparingly. Do not use emojis in headers.

## 4. Publishing Workflow
1. **Draft**: Create your content in the "Draft" state.
2. **Preview**: Use the "Preview" button to see how it looks on the staging site.
3. **Review**: Ensure all links work and no typos exist.
4. **Publish**: Click "Publish" to push changes to the live site.
