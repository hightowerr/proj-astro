# Problem Diagnosis: MVP for Promo Code Validation

The user wants to define the simplest possible user journey to validate the core assumption that "users will value the ability to apply a promo code in the basket."

This is an exercise in defining a Minimum Viable Product (MVP) or experiment. The goal is to find the path of maximum learning for the minimum amount of development effort.

The analysis needs to focus on identifying the absolute minimum set of features required to test this hypothesis effectively. It should consider:

1.  **The Core Hypothesis:** What specific value are we testing? (e.g., convenience, seeing savings, etc.)
2.  **The Simplest User Interaction:** What is the most basic way a user can perform the core action?
3.  **The Minimum Feedback Loop:** What is the simplest way to show the user the result of their action and for us to measure their "value" perception? (e.g., successful application message, seeing the discount).
