# Occam's Razor

**Description:**
Occam's Razor is a principle stating that among competing hypotheses that predict the same outcome, the one with the fewest assumptions should be selected. Named after William of Ockham, this mental model suggests that simpler explanations are generally better than complex ones, all else being equal. It helps cut through complexity and avoid unnecessary complications when simpler solutions are available. This doesn't mean the simplest answer is always right, but it's usually the best starting point.

**When to Avoid (or Use with Caution):**
- When oversimplifying complex situations that require nuanced understanding
- When the simple explanation doesn't account for all relevant facts
- When complex problems legitimately require complex solutions
- When using simplicity as an excuse to avoid difficult analysis
- When the apparently simple solution ignores important variables or relationships

**Keywords for Situations:**
Problem solving, decision making, explanation, theory building, analysis, complexity, simplicity, troubleshooting, research, diagnosis.

**Thinking Steps:**
1. **Formulate Hypotheses:** Create multiple explanations for the same phenomenon.
2. **Count Assumptions:** Count how many assumptions each explanation requires.
3. **Start Simple:** Begin with the explanation requiring the fewest assumptions.
4. **Test Simpler Models:** Validate simpler explanations before considering complex ones.
5. **Add Complexity Selectively:** Only add complexity when simpler models don't adequately explain observations.
6. **Balance Simplicity with Accuracy:** Ensure the simple model captures essential features of the situation.

**Coaching Questions:**
* "What is the simplest explanation that accounts for all the observed phenomena?"
* "Am I making more assumptions than necessary?"
* "Will a simpler explanation work as well as the complex one I'm considering?"
* "What essential elements must be preserved if I simplify this explanation?"
* "Could I be oversimplifying a complex situation?"
* "How might I test whether a simpler explanation is sufficient?"
