# Logic Trees (Issue Trees)

**Description**: A core tool for problem disaggregation (Step 2 of the Bulletproof Problem Solving framework). A logic tree is a visual representation that breaks a complex problem down into its logical, component parts. This helps to ensure the analysis is holistic (all relevant parts are captured), provides clarity for the entire team, and leads to clear, testable hypotheses. The book highlights several types, including hypothesis trees, decision trees, and profit lever trees.

**When to Avoid (or Use with Caution)**: A poorly constructed tree can lead the analysis astray. It's important that the branches of the tree are as MECE (Mutually Exclusive, Collectively Exhaustive) as possible to avoid confusion and overlapping work. The tree is a map of the problem, not the solution itself; rigorous analysis is still required to validate the hypotheses the tree helps to generate. For very simple problems, a formal tree may be unnecessary.

**Keywords for Situations**: Problem structuring, root cause analysis, brainstorming, strategic planning, case interviews, quantitative analysis, decision making.

**Thinking Steps**:
1.  Start with the core problem statement as the root of the tree (e.g., "Will Sydney Airport capacity be adequate in the future?").
2.  Break the problem down into its first-level components. This "cleave" is the most critical step. For a capacity problem, it might be `Capacity Gap = Demand - Supply`. For a business problem, it might be `Profit = Revenue - Costs`.
3.  Continue to break down each component into smaller, more detailed sub-components until you have isolated the specific, analyzable drivers of the problem (e.g., breaking down `Supply` into `Runways * Hours of Operation * Planes per Hour`).
4.  Formulate a testable hypothesis for the branches that seem most critical to solving the problem.
5.  Use the finished tree structure to guide your workplan, assigning specific branches to team members for analysis.

**Coaching Questions**:
*   "How can we visually break this problem down? Let's try to build a logic tree on the whiteboard."
*   "Are the branches of our tree mutually exclusive and collectively exhaustive? Have we missed anything?"
*   "Which branch of this tree, if we could prove or disprove it, would have the biggest impact on our overall answer?"
*   "Does this tree lead us to clear hypotheses that we can now go and test with data?"
