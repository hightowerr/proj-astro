## Mental Model = Jevons Paradox

**Category = Economics / Systems Thinking**
**Description:**
Jevons Paradox is the counterintuitive pattern where improving efficiency in using a resource can increase (not decrease) total consumption of that resource. The core mechanism is economic and behavioral: efficiency lowers the cost of producing/consuming a valued service, so usage expands, new applications become viable, and system scale grows.

**When to Avoid (or Use with Caution):**
- When demand is truly saturated and cannot expand materially.
- When strict caps/prices/quotas bind at the system level (so savings don’t translate into more use).
- When you only need a short-run, local analysis and system-scale feedbacks are irrelevant.

**Keywords for Situations:**
Energy policy; sustainability; productivity; “paperless office”; traffic; “green growth”; cost reduction; automation; resource constraints.

**Thinking Steps:**
1. Define the resource and the service it provides (what’s getting “cheaper”?).
2. Separate intensity vs total: per-unit use vs aggregate use.
3. Identify demand elasticities and expansion paths (more of the same use, new uses, scale-up).
4. Trace indirect effects (income effects, substitution, sectoral shifts, trade).
5. Ask what constrains total consumption (prices, caps, norms, physical limits).
6. Design policy/strategy at the system level (caps, pricing, budgets, governance), not only efficiency.

**Coaching Questions:**
- If this becomes cheaper/easier, what will people do more of?
- What new uses become viable at the new cost?
- Are we measuring “per unit” improvements while missing total growth?
- What would keep total consumption from rising (and is it enforceable)?
- If we want absolute reduction, where is the binding constraint/cap?
