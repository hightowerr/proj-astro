# Mental Model: Problem Finding

## Description
Problem finding is the process of identifying and formulating the right problem to solve, which is often more critical than finding a solution. It involves going beyond the initially presented problem to discover the true underlying issue. Unlike problem solving, which assumes a problem is already defined, problem finding is about creating the problem statement itself. This mental model emphasizes that most problems have multiple causes and can be addressed in many ways, challenging the assumption that there is one "real" problem to solve.

## When to Avoid (or Use with Caution)
Problem finding should be used with caution when:
- Time constraints require immediate action
- The problem is clearly identified and has a proven solution
- Stakeholders are not open to questioning the initial problem framing
- There is insufficient information to explore alternative problem definitions
- The situation requires rapid response rather than systematic exploration
- Resources are extremely limited and cannot support problem exploration
- The problem is straightforward with clear cause-and-effect relationships

## Keywords for Situations
- When teams are jumping immediately to solution mode without questioning the problem
- When existing solutions to a problem haven't been effective
- When stakeholders have different understandings of what the problem is
- When facing complex organizational challenges with multiple contributing factors
- When customers complain about outcomes without clearly articulating their core need
- When solutions from other organizations aren't working in your context
- When there's disagreement about the root cause of an issue
- When facing recurring issues that get temporarily fixed but return

## Thinking Steps
1. Write down the problem as it's currently stated
2. Question whether this is the real problem or just how it's been presented
3. Look for evidence that contradicts the initial problem statement
4. Identify stakeholders who might see this differently and their perspectives
5. Ask "why" multiple times to get to deeper causes
6. Consider what might happen if the stated problem disappeared - would there still be an issue?
7. Look for positive exceptions where the problem doesn't occur
8. Test your alternative problem statements against available data

## Coaching Questions
- What if the problem we're trying to solve isn't actually the real problem?
- How might different stakeholders define this situation as a problem?
- What are we assuming is the "real" problem that might be incorrect?
- Where does this problem NOT exist? What's different in those contexts?
- If we solved the stated problem, what else would we need to address?
- What would happen if we stopped trying to solve this problem immediately?
- Who else might be affected by this issue that we haven't considered?
- What evidence do we have that our current problem definition is accurate?
