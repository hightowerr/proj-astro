# Mental Model: Exploring vs Breaking the Frame

## Description
Exploring vs Breaking the Frame describes two different approaches to reframing problems. Exploring the frame involves delving deeper into the original problem statement, looking for overlooked aspects that might make a difference. Breaking the frame means stepping away completely from the initial framing of the problem and putting an entirely different spin on it. Both approaches can lead to breakthroughs, but breaking the frame is more important as it prevents getting trapped by the initial problem framing.

## When to Avoid (or Use with Caution)
Use with caution when:
- The problem is well-defined with clear parameters
- Immediate action is required and there's no time for exploration
- Stakeholders are not open to changing the problem framing
- The existing frame is already accurate and productive
- Resources are extremely limited for problem exploration
- You're implementing a standard procedure with known problems
- The team is experiencing decision fatigue and needs to focus

## Keywords for Situations
- When facing persistent problems that haven't been solved by conventional approaches
- When stakeholders are entrenched in a particular problem view
- When solutions within the current frame keep failing
- When different stakeholders have conflicting views of the problem
- When encountering opportunities that don't fit the current problem frame
- When trying to innovate beyond existing solutions
- When dealing with complex systems where multiple problems exist
- When traditional analysis isn't yielding effective solutions

## Thinking Steps
1. Start with the initial problem framing given to you
2. Try exploring the frame first - dig deeper into the stated problem to find hidden aspects
3. If exploring doesn't yield insights, try breaking the frame entirely
4. Ask "What if we looked at this from a completely different angle?"
5. Identify what assumptions are embedded in the current frame
6. Consider what would happen if those assumptions were reversed
7. Look for stakeholders or perspectives that might see the situation differently
8. Test both explored and broken frames against the evidence and stakeholder needs

## Coaching Questions
- Are we focusing too much on the details of the current problem instead of questioning the frame itself?
- What happens if we completely disregard the way this problem has been presented to us?
- Who else might see this situation as a completely different kind of problem?
- What assumptions are we making about this situation that might be wrong?
- If we had to explain this situation to someone with no knowledge of the context, how would we frame it?
- What would happen if we inverted our current problem statement?
- Are there stakeholders we haven't consulted whose problem might be different from ours?
- What if the problem we're trying to solve is actually a symptom of something else?
