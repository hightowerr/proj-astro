## Mental Model = Ceteris Paribus Assumption

**Category = Problem Solving / Systems Thinking**
**Description:**
The ceteris paribus (“all else equal”) assumption isolates the effect of one variable by holding others constant. It can clarify local mechanisms, but it becomes misleading when systems adapt: preferences shift, behaviors change, new constraints bind, and feedback loops dominate.

**When to Avoid (or Use with Caution):**
- When the intervention changes incentives, prices, or constraints (so people adapt).
- When the system operates across multiple scales (micro → macro feedback).
- When you’re using it to claim long-run total outcomes rather than short-run local effects.

**Keywords for Situations:**
Forecasting; econometrics; policy evaluation; efficiency; demand response; systems thinking.

**Thinking Steps:**
1. State explicitly what is being held constant (and why).
2. List the most likely adaptation channels if those variables are not constant.
3. Bound the claim: short-run/local vs long-run/system-wide.
4. Stress-test with at least one scenario where “all else” changes.
5. Upgrade the model boundary if adaptation is central to the question.

**Coaching Questions:**
- What are we assuming stays constant—and is that plausible after this change?
- Which actors can adapt, and how quickly?
- What feedback loops could amplify or dampen the effect we modeled?
